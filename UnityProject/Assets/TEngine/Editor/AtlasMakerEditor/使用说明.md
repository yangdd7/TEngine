# AtlasMaker 编辑工具使用说明

## 工具概述

AtlasMaker 是一个 Unity 编辑器扩展工具，用于**自动生成和管理精灵图集 (Sprite Atlas)**。它能够：

- 📦 自动扫描指定目录下的图片文件
- 🎨 自动生成对应的精灵图集
- 🔄 监听文件变化，实时更新图集
- ⚙️ 灵活配置多种生成策略
- 🚀 支持 Unity 2022.1+ 的 SpriteAtlasV2 格式

---

## 快速开始

### 1. 打开配置面板

在 Unity 编辑器菜单栏中选择：
```
Tools → 图集工具 → 配置面板
```

会打开一个名为 "图集配置窗口" 的编辑器窗口。

### 2. 配置基本设置

#### 目录设置（第一部分）

这是最重要的配置，决定了图集的生成逻辑。

**输出目录**
- 图集生成后保存的目录
- 默认值：`Assets/AssetArt/Atlas`
- 点击 "选择" 按钮可以图形化选择文件夹

**收集目录**
- 指定需要生成图集的源图片目录
- 默认值：`Assets/AssetRaw/UIRaw/Atlas`
- 可以配置多个目录
- 点击 "数量" 旁的 **+/-** 按钮增减目录数量
- 该目录下的文件夹结构会直接对应图集名称

**⚠️ 收集目录配置的重要注意事项：**

1. **目录必须存在**
   - 配置的目录必须在项目中实际存在
   - 如果目录为空，添加时不会报错，但工具会跳过该目录
   - 在 ProjectSettings/AtlasConfiguration.asset 中手动修改时需特别注意

2. **路径格式要求**
   - 必须以 `Assets/` 开头
   - 路径间使用 `/` 分隔符（不要用 `\`）
   - 不要在路径末尾添加 `/`，工具会自动处理
   - 示例正确格式：`Assets/AssetRaw/UIRaw/Atlas`
   - 示例错误格式：`AssetRaw/UIRaw/Atlas`、`Assets/AssetRaw/UIRaw/Atlas/`

3. **配置多个目录的注意事项**
   - ✅ 让不同的收集目录中的文件夹结构尽量不同，避免生成相同名称的图集
   - ❌ 不要在多个收集目录中配置相同的子路径
   - 多个目录中**不能有包含关系**（例如不能同时配置 `Assets/AssetRaw/UI` 和 `Assets/AssetRaw/UI/Dialog`）
   - 如果有包含关系，工具会使用第一个匹配的目录，后面的会被忽略

4. **单张图集与平级目录的关系**
   - 如果要在新添加的收集目录中使用 "单张图生成图集" 功能
   - 必须先将该新目录配置在 **收集目录** 中
   - 然后在 **单张图生成图集的目录** 中配置该收集目录的子目录
   - 例如：
     ```
     收集目录[0] = Assets/AssetRaw/UIRaw/Atlas
     收集目录[1] = Assets/AssetRaw/GameRaw/UI  ← 新添加的目录
     
     单张图生成图集的目录[0] = Assets/AssetRaw/UIRaw/Atlas/Background
     单张图生成图集的目录[1] = Assets/AssetRaw/GameRaw/UI/Avatar   ← 基于新收集目录
     ```

5. **配置修改后的处理**
   - 修改收集目录配置后，可能需要清空缓存以使新配置生效
   - 步骤：点击底部 "清空缓存" 按钮 → 点击 "重新生成有变动的图集数据"
   - 如果仍无效，点击 "立即重新生成" 按钮（会删除所有图集并重新生成）

6. **排除目录的优先级**
   - 如果一个目录同时被配置为收集目录和排除目录，排除目录优先级更高
   - 该目录中的文件不会被处理

**排除目录**
- 指定需要被排除的目录（这些目录下的图片不会生成图集）
- 默认值：`Assets/AssetRaw/UIRaw/Raw`
- 可以配置多个目录

**根目录子级生成图集**
- 只针对指定根目录的第一级子文件夹生成图集
- 默认值：空
- 适合按类别生成图集的场景（如：Button、Panel 等）

**单张图生成图集**
- 指定目录下的每张图片都单独生成一个图集
- 默认值：`Assets/AssetRaw/UIRaw/Atlas/Background`
- 适合大贴图或特殊需求的图片

## 详细配置说明

### 平台设置

配置不同平台的纹理格式和压缩质量。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| Android 格式 | Android 平台的纹理压缩格式 | ASTC_6x6 |
| iOS 格式 | iOS 平台的纹理压缩格式 | ASTC_5x5 |
| WebGL 格式 | WebGL 平台的纹理压缩格式 | ASTC_6x6 |
| 压缩质量 | 纹理压缩质量（0-100） | 50 |

**推荐设置：**
- **移动平台**：使用 ASTC 格式（更好的压缩比）
- **高质量需求**：提高压缩质量到 70-80
- **低配设备**：降低压缩质量到 30-40

### 图集设置

控制精灵图集的打包参数。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| Padding | 相邻图片间的间距（像素） | 2 |
| Block Offset | 块偏移值 | 1 |
| Enable Rotation | 是否允许旋转优化 | ✓ 启用 |
| 剔除透明区域 | 是否开启紧密打包 | ✓ 启用 |

**参数说明：**
- **Padding**：增大可以避免纹理采样时的边界渗色问题
- **Enable Rotation**：启用后可能更省空间，但需要 Shader 支持
- **剔除透明区域**：自动移除透明边界，可节省纹理空间

### Sprite 导入设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 检查 Mipmap 导入设置 | 是否检查 Mipmap 设置 | ✓ 启用 |
| 允许 Mipmap | 是否生成 Mipmap | ✗ 禁用 |

**Mipmap 说明：**
- UI 图集通常**不需要** Mipmap
- 3D 模型纹理**需要** Mipmap

### 高级设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 自动生成 | 文件变更时是否自动生成图集 | ✓ 启用 |
| 启用日志 | 是否输出操作日志到控制台 | ✓ 启用 |
| 启用 V2 打包 | 使用新的 SpriteAtlasV2 格式 | ✓ 启用 |

**排除关键词**
- 文件名包含这些关键词的图片会被排除（不生成图集）
- 默认值：`_Delete`、`_Temp`
- 不区分大小写
- 点击 "数量" 旁的 **+/-** 按钮增减关键词数量

## 图集命名规则

工具根据目录结构自动生成图集名称。

### 规则

```
图集名 = 根文件夹名 + "_" + 子目录层级路径
```

### 示例

假设收集目录为 `Assets/AssetRaw/UIRaw/Atlas`

| 图片路径 | 生成的图集 | 说明 |
|----------|----------|------|
| `...Atlas/Button/btn_1.png` | `UIRaw_Button.spriteatlas` | 二级目录 |
| `...Atlas/Dialog/Panel/img.png` | `UIRaw_Dialog_Panel.spriteatlas` | 三级目录 |
| `...Atlas/img.png` | 不生成（排除） | 一级目录下的文件被排除 |

### 特殊命名道则

#### 单张图集
如果图片在 `单张图生成图集的目录` 配置中：
```
图集名 = 根文件夹名 + "_" + 文件夹路径 + "_" + 文件名
```

#### 根目录子级图集
如果目录在 `以根目录的子级目录生成图集` 配置中：
```
图集名 = 根文件夹名 + "_" + 第一级子目录名
```

## 常见问题

### Q1：图片文件名不能含有空格
**现象**：导入包含空格名字的图片时报错
```
<color=red>发现资源名存在空格: Assets/.../my image.png</color>
```

**解决方案**：
- 将图片重命名，去掉空格
- 示例：`my_image.png`、`myImage.png`

### Q2：配置多个收集目录时，新添加的目录无效
**现象**：
- 在默认值的平级目录中添加新目录
- 该目录中的图片没有被处理，没有生成对应的图集
- 新目录中的文件变化没有被识别

**原因分析**：
1. **最常见原因**：新目录不存在或路径不正确
   - 检查路径是否真的存在于项目中
   - 确认路径必须以 `Assets/` 开头
   - 不要使用反斜杠 `\`

2. **次常见原因**：新目录与排除目录冲突
   - 检查新目录是否被配置在 "排除目录" 中
   - 检查新目录是否包含排除关键词（默认 `_Delete`、`_Temp`）

3. **缓存问题**：缓存未更新
   - 配置修改后，工具的缓存可能未及时更新

**解决步骤**（按顺序尝试）：

**步骤 1：验证目录配置**
```
1. 打开配置面板
2. 展开"收集目录"
3. 验证新添加目录的路径是否：
   - 正确（必须以 Assets/ 开头）
   - 存在（在项目中可以找到该文件夹）
   - 没有末尾的 /（例如 Assets/AssetRaw/UI/ ❌ → Assets/AssetRaw/UI ✅）
```

**步骤 2：检查排除规则**
```
1. 检查 "排除目录" 配置
   - 新目录不能被排除目录包含
   - 例如：不能同时配置 Assets/AssetRaw/UI 作为收集目录
             和 Assets/AssetRaw 作为排除目录

2. 检查 "排除关键词"
   - 新目录中的文件名不能包含 _Delete、_Temp 等关键词
3. 检查排除关键词列表是否有无意中添加的关键词
```

**步骤 3：清空缓存并重新生成**
```
1. 点击底部 "清空缓存" 按钮
2. 等待操作完成
3. 点击 "重新生成有变动的图集数据" 按钮
4. 监控 Console 输出，查看新目录中的文件是否被处理
```

**步骤 4：完全重建（最后手段）**
```
1. 确保新添加的目录中已有图片文件
2. 点击底部 "立即重新生成" 按钮（黄色）
3. 弹出确认对话框，点击 "删除"
4. 等待完整的重新生成过程

💡 提示：启用 "启用日志" 可以看到详细的处理日志
```

**验证是否成功**：
```
1. 打开 Console（Ctrl+Shift+C）
2. 在搜索栏搜索新目录中的文件名
3. 查看是否有 "[Added]" 或 "[Generate Atlas]" 的日志
4. 检查 Assets/AssetArt/Atlas 中是否生成了对应的图集
```

**示例演示**：

假设你要添加 `Assets/AssetRaw/GameUI/Atlas` 作为新的收集目录

❌ **错误操作**：
```
收集目录[1] = Assets/AssetRaw/GameUI/Atlas/
                                        ❌ 末尾有 /
                                        
结果：目录可能被跳过
```

✅ **正确操作**：
```
收集目录[0] = Assets/AssetRaw/UIRaw/Atlas
收集目录[1] = Assets/AssetRaw/GameUI/Atlas    ✅ 无末尾 /

排除目录 = 不包含 Assets/AssetRaw/GameUI
排除关键词 = 无与 GameUI 中文件名冲突的关键词

然后在 Console 中验证：
[Added] btn_start.png
Path: Assets/AssetRaw/GameUI/Atlas/Button/btn_start.png

[Generate Atlas]: GameUI_Button (1 sprites)
```

### Q3：发现同名资源冲突
**现象**：包含以下错误消息
```
<color=red>发现同名资源冲突: 合入资源: ... 存在资源: ...</color>
```

### Q3：发现同名资源冲突
**现象**：包含以下错误消息
```
<color=red>发现同名资源冲突: 合入资源: ... 存在资源: ...</color>
```

**原因**：
- 配置的多个目录中存在同名文件

**解决方案**：
- 检查收集目录配置
- 重命名冲突的文件
- 或调整目录配置避免重复

### Q4：自动生成没有反应
**检查项**：
1. 确认 "自动生成" 已勾选
2. 确认图片在配置的收集目录内
3. 确认没有被排除目录或关键词排除
4. 检查是否有权限写入输出目录

### Q5：图集文件体积过大
**优化方案**：
1. 增大 Padding 参数，启用 "剔除透明区域"
2. 提高 "压缩质量" 的值（权衡文件大小和质量）
3. 使用 ASTC 格式而不是 RGB 格式
4. 考虑分多个小图集而不是一个大图集

### Q6：如何禁用自动生成？
**方案**：
- 去掉 "自动生成" 的勾选
- 手动点击 "重新生成有变动的图集数据" 按钮

### Q7：如何查看操作日志？
**步骤**：
1. 勾选 "启用日志"
2. 在 Unity 控制台 (Console) 中查看
3. 搜索 "[Generate Atlas]" 或 "[Added]"、"[Deleted]" 关键词

## 多个收集目录的工作原理

### 内部处理逻辑

当配置多个收集目录时，工具会按照以下流程处理：

```
1. 用户导入/修改图片
   ↓
2. AssetPostprocessor 捕获文件变化
   ↓
3. 检查该图片的路径是否在任一个收集目录中
   └─ if (assetPath.StartsWith(收集目录[0] + "/")) → 使用收集目录[0]
   └─ if (assetPath.StartsWith(收集目录[1] + "/")) → 使用收集目录[1]
   └─ ... （按顺序检查）
   └─ if 都不匹配 → 跳过处理
   ↓
4. 提取相对路径生成图集名称
   ├─ 根文件夹名（收集目录[0]） + "_" + 文件夹路径
   ├─ 例如：UIRaw_Button、GameUI_Dialog
   ↓
5. 生成或更新图集
```

### 路径匹配规则（关键！）

代码中的路径检查使用了 **前缀匹配**：

```csharp
if (!assetPath.StartsWith(tempPath + "/"))
{
    continue;  // 不匹配，检查下一个目录
}
```

这意味着：
- 路径必须**完全匹配**
- `Assets/AssetRaw/UI` ✅ 会匹配收集目录 `Assets/AssetRaw/UI`
- `Assets/AssetRaw/UI_Temp` ❌ 不会匹配收集目录 `Assets/AssetRaw/UI`（因为没有 `/` 分隔符）
- `Assets/AssetRaw/UI/` ❌ 收集目录不能以 `/` 结尾

### 包含关系问题

如果配置了有包含关系的目录，**只有第一个会生效**：

```
收集目录[0] = Assets/AssetRaw/UI         ← ✅ 有效
收集目录[1] = Assets/AssetRaw/UI/Dialog  ← ❌ 无效（被包含）

原因：Dialog 中的文件会在第一次检查时被 UI 匹配，不会继续检查第二个
```

**正确做法**：
```
收集目录[0] = Assets/AssetRaw/UIRaw/Atlas
收集目录[1] = Assets/AssetRaw/GameUI/UI
↑                          ↑
不同的顶级目录，无包含关系，都会有效
```

### 单张图集与根目录子级的限制

这两个配置有一个重要限制：**它们必须建立在收集目录的基础上**

```
收集目录 = 【基础】
    ↓
单张图集目录 = 【收集目录内的子目录】
根目录子级图集目录 = 【收集目录内的子目录】

如果配置的是收集目录外的其他目录，这些特殊规则不会应用
```

**错误示例**：
```
收集目录[0] = Assets/AssetRaw/UI
             
单张图集目录 = Assets/AssetRaw/Other/Avatar  ← ❌ 不在收集目录内，无效

原因代码：
private static string GetSingleAtlasName(string spritePath)
{
    foreach (var rootPath in Config.sourceAtlasRootDir)  ← 只遍历收集目录
    {
        if (!spritePath.StartsWith(rootPath + "/"))
        {
            continue;  ← Other 不在收集目录内，会跳过
        }
        // ...生成单张图集
    }
}
```

**正确示例**：
```
收集目录[0] = Assets/AssetRaw/UI
             
单张图集目录 = Assets/AssetRaw/UI/Avatar  ← ✅ 在收集目录内，有效
```

### 文件监听的工作机制

```
文件变化 → 捕获 → 检查目录匹配 → 扫描所有收集目录 → 对比文件

ScanExistingSprites() 会同时在所有收集目录中查找精灵文件：

var guids = AssetDatabase.FindAssets("t:sprite", Config.sourceAtlasRootDir);
                                                   ↑ 数组，包含所有收集目录

这意味着只要其中一个目录配置正确，就能被找到
```

// 清空工具缓存（仅编辑器下有效）
EditorSpriteSaveInfo.ClearCache();
```

### 技巧 3：使用排除关键词过滤图片
在高级设置中添加排除关键词：
- `_Delete` - 标记为删除的图片
- `_Temp` - 临时图片
- `_UI` - 某类 UI 图片

```
.gitignore 中建议忽略以下文件：
Atlas/*.spriteatlas
Atlas/*.spriteatlasv2
Atlas/*.spriteatlas.meta
Atlas/*.spriteatlasv2.meta

## 收集目录配置检查清单

如果配置多个收集目录后无效，请按以下清单一项一项检查：

### 路径格式检查
- [ ] 路径以 `Assets/` 开头
- [ ] 路径不包含反斜杠 `\`（只有 `/`）
- [ ] 路径末尾没有 `/` 符号
- [ ] 路径中没有多余的空格
- [ ] 文件夹在项目中实际存在

### 配置逻辑检查
- [ ] 多个收集目录之间没有包含关系
  - ❌ 同时配置 `Assets/UI` 和 `Assets/UI/Button`
  - ✅ 配置 `Assets/UIRaw` 和 `Assets/GameUI`

- [ ] 新目录不被其他配置规则排除
  - [ ] 检查 "排除目录" 是否包含该目录
  - [ ] 检查 "排除关键词" 是否与文件名冲突

- [ ] 如果使用了 "单张图集" 或 "根目录子级"
  - [ ] 这些目录必须在某个收集目录内
  - [ ] 不能独立配置在项目的其他位置

### 工具状态检查
- [ ] "自动生成" 已勾选（如果需要自动生成）
- [ ] 新目录中已有实际的图片文件（不是空文件夹）
- [ ] 已清空缓存（点击 "清空缓存" 按钮）

### 验证检查
- [ ] 启用 "启用日志" 并打开 Console
- [ ] 在新目录中放入一张图片
- [ ] 观察 Console 是否出现处理日志
- [ ] 检查输出目录中是否生成了图集文件

### 终极解决方案
如果上述所有检查都通过仍然无效：
1. [ ] 点击 "清空缓存"
2. [ ] 点击 "立即重新生成"（注意：会删除所有现有图集）
3. [ ] 在 Console 中验证是否成功

## 最常见的错误场景和解决方案

### 场景 1：收集目录路径末尾有 `/`

❌ **错误配置**：
```
收集目录[1] = Assets/AssetRaw/GameUI/
                                 ↑ 末尾有 /
```

✅ **正确改法**：
```
收集目录[1] = Assets/AssetRaw/GameUI
                                 ↑ 已移除 /
```

然后点击"清空缓存" → "重新生成有变动的图集数据"

---

### 场景 2：新目录与默认目录有包含关系

❌ **错误配置**：
```
收集目录[0] = Assets/AssetRaw/UIRaw/Atlas
收集目录[1] = Assets/AssetRaw/UIRaw/Atlas/DialogUI
            ↑ 被包含在 [0] 中，会被第一个匹配
```

✅ **正确改法**：
```
收集目录[0] = Assets/AssetRaw/UIRaw/Atlas
收集目录[1] = Assets/AssetRaw/GameRaw/UI  ← 其他分支，不包含关系
```

---

### 场景 3：单张图集配置外层目录

❌ **错误配置**：
```
收集目录[0] = Assets/AssetRaw/UI

单张图集目录[0] = Assets/AssetRaw  ← 不在收集目录内
       ↑ 这个会被忽略，不会生成单张图集
```

✅ **正确改法**：
```
收集目录[0] = Assets/AssetRaw/UI

单张图集目录[0] = Assets/AssetRaw/UI/Avatar  ← 在收集目录内
       ↑ 这样就能正常工作了
```

---

### 场景 4：排除关键词意外匹配

❌ **问题**：
```
收集目录[1] = Assets/AssetRaw/GameUI
排除关键词 = ["_Delete", "Game"]  ← "Game" 作为关键词
           
GameUI 中的所有文件都会被排除（因为包含 "Game"）
```

✅ **解决**：
```
排除关键词 = ["_Delete", "_Temp", "_old"]  ← 更具体的关键词
           
这样就不会误伤 GameUI 了
```

## 故障排查

### 问题诊断流程

1. **检查日志**
   - 勾选 "启用日志"
   - 查看 Unity Console 输出

2. **验证配置**
   - 确认目录路径是否正确
   - 检查路径是否存在

3. **清空缓存**
   - 点击 "清空缓存" 按钮
   - 重新生成图集

4. **重新导入**
   - 在 ProjectSettings/AtlasConfiguration.asset 上右键
   - 选择 "Reimport"

### 常见错误信息

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| 路径不在Unity项目内 | 选择了项目外的文件夹 | 重新选择正确的路径 |
| 发现资源名存在空格 | 图片文件名含有空格 | 重命名去掉空格 |
| 发现同名资源冲突 | 有重名文件 | 重命名其中一个 |
